# 📊 Data Quality Monitor

Система автоматического мониторинга качества данных в распределённых информационных системах с применением статистических методов контроля процессов (SPC).

Разработана в рамках выпускной квалификационной работы 2026 года.

---

## 🔍 О проекте

Система выявляет деградацию качества данных в реальном времени — без участия аналитика и без знания статистики. Агенты мониторинга подключаются к источникам данных, вычисляют метрики качества и отправляют их на центральный сервер, который применяет три статистических метода:

- **Карта Шухарта** — обнаруживает резкие выбросы за ±3σ
- **CUSUM** — накапливает малые устойчивые отклонения, невидимые Шухарту
- **EWMA** — сглаживает тренд и показывает направление изменения

Каждый метод решает свою задачу — вместе они покрывают все основные виды деградации данных.

## ⚙️ Установка и запуск

### 1. Требования

- Python 3.13+
- PostgreSQL 18+

### 2. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 3. Настройка базы данных

Создайте две базы данных в PostgreSQL:

```sql
CREATE DATABASE data_quality_metrics;  -- для метрик системы
CREATE DATABASE source_a;              -- источник данных A
```

Если нужно — измените параметры подключения в `server/database.py` и `agents/agent_a.py`.

### 4. Запуск сервера

```bash
uvicorn server.main:app --reload
```

Сервер запустится на `http://127.0.0.1:8000`. Документация API доступна по адресу `http://127.0.0.1:8000/docs`.

### 5. Запуск веб-интерфейса

В отдельном терминале:

```bash
streamlit run dashboard/app.py
```

Интерфейс откроется на `http://localhost:8501`.

## 🧪 Проведение эксперимента

Откройте страницу **Управление экспериментом** и настройте параметры. Система сама сгенерирует данные, загрузит их в источники и запустит агентов.

### Рекомендуемые параметры для первого запуска

| Параметр | Значение |
|---|---|
| Базовое среднее (μ) | 100 |
| Базовое стд. откл. (σ) | 5 |
| Размер окна агента | 20 |
| Задержка между окнами | 1 сек |

**Источник A — PostgreSQL:**

| Параметр | Значение |
|---|---|
| Записей | 300 |
| Вид деградации | Постепенный дрейф |
| Начало деградации | запись №100 |
| Дрейф за шаг | 0.3 |
| Метрики | Среднее, Медиана |

**Источник B — SQLite:**

| Параметр | Значение |
|---|---|
| Записей | 300 |
| Вид деградации | Пропуски |
| Начало деградации | запись №100 |
| Доля пропусков | 0.3 |
| Метрики | Полнота, Среднее |

**Источник C — CSV:**

| Параметр | Значение |
|---|---|
| Записей | 300 |
| Вид деградации | Случайные выбросы |
| Начало деградации | запись №100 |
| Вероятность выброса | 0.15 |
| Метрики | Среднее, Стд. отклонение, МКР (IQR) |

После запуска перейдите на **Дашборд** — система обнаружит деградацию и отобразит сигналы по каждому источнику. Детали доступны на странице **Детальный анализ**.

## 📈 Поддерживаемые виды деградации

| Вид | Описание |
|---|---|
| Без деградации | Нормальный режим, только случайный шум |
| Сдвиг среднего | Резкий скачок μ на заданную величину Δ |
| Рост дисперсии | Стандартное отклонение увеличивается на Δσ |
| Пропуски | Доля NaN-значений растёт после заданной записи |
| Постепенный дрейф | Среднее смещается линейно на каждом шаге |
| Случайные выбросы | Отдельные значения резко отклоняются с заданной вероятностью |
